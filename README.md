# Event Driven Backtesting Engine for Options 

 A high-performance event-driven backtesting engine written in C++. It is designed to simulate and analyze options trading strategies with a primary focus on an extensible architecture. 

 Scripting is done via the Python API made through `pybind11` which exposes the engine as a  module, tasks like data processing, option pricing, and execution simulation—is handled natively in C++.

## Required CSV Data Format & Data Handling
The engine processes historical option market data via third-party integrations (only [Databento](https://databento.com/) for now ) or CSV files. When using CSV files, the data format must encompass standard option metrics along with specific required fields. Other than that if using databento 
requested data will get cached first in `data/options/tickername`.


**Required CSV Fields Structure:**
- `timestamp (dd-mm-yy)`: Date and time of the market data.
- `symbol`: The unique identifier of the option contract.
- `underlying_price`: The spot price of the underlying asset.
- `strike`: The strike price of the option.
- `option_type`: Call or Put.
- `expiry (dd-mm-yy)`: The expiration date of the contract.
- `bid` / `ask` / `close`: Pricing information.
- `volume`: Traded volume.
- `risk_free_rate`: The prevailing risk-free interest rate.
- `dividend_yield`: The dividend yield of the underlying asset.

> **Note on Dividend_yield and rfr :** Although `dividend_yield` and `risk_free_rate` are required to be present in the CSV for the engine to parse the data correctly, they are not actively utilized in the current pricing models. As of right now, they are internally hardcoded to `0` and `5` respectively.

## Events in the Backtester
The engine's simulation revolves around an event-driven architecture, processing the following key events:

- **`MarketEvent`**: Triggered when a new market data snapshot (`OptionChain`) arrives. Strategies listen to this event to analyze the market and generate trading signals.

- **`SignalEvent`**: can be Generated by a `Strategy` in response to a `MarketEvent`. It indicates a desire to trade a specific option contract (e.g., Long, Short, Exit).

- **`FillEvent`**: Simulated by the engine after processing a `SignalEvent`. It details the executed trade, including the fill price, quantity, slippage, and commissions incurred.

## Prerequisites & Installation
To build the C++ core and the Python bindings, the following dependencies are required:
- **CMake** (v3.15+)
- **Compiler**: The project is configured to be compiled using **MSVC** (Microsoft Visual C++).

**If you are using a different compiler (like GCC or Clang), you will need to tweak the CMakeLists.txt accordingly.**

- **OpenSSL** *need to be installed internally and must be accessible by cmake using -DCMAKE_TOOLCHAIN_FILE*
- **zstd** *need to be installed internatlly and must be accessible by cmake using -DCMAKE_TOOLCHAIN_FILE (I used vcpkg for both)* 
- **nlohmann_json** *comes with databento sdk*
- **httplib** *comes with databento sdk*
- **pybind11**
- **pybind11 - stubgen** *(this is optional if using specific tools - might already provide intellisense)*

*(Note: OpenSSL, zstd, nlohmann_json, and httplib are dependencies needed specifically for the Databento SDK .)*

**Build Steps:**
1. Clone the repository with the `--recurse-submodules` flag .

```bash
git clone --recurse-submodules https://github.com/vaibhavrohilla-03/Option-Strategy-Backtesting-Engine.git
```
2. Create a build directory: `mkdir build && cd build`
3. Generate the build files: `cmake ..`
4. Build the project: `cmake --build . --config Release`
5. The output will be a shared library which will be outputted in the strategies folder (`py_backtester.pyd` on Windows or `.so` on Linux) that you can import directly in Python.

## Getting Started / Quick Start
Below is a minimal Python example demonstrating how to implement a simple Bull Call Spread strategy using the engine. 
An API reference doc is provided 

```python
import py_backtester as bt
from datetime import date

class BullCallSpread(bt.Strategy):
    def __init__(self):
        super().__init__("BullCallSpread") #needed for parent class 
        self.last_trade_date = None

    def on_event(self, event): # overriden method that will be called each event 
        
        if event.type == bt.EventType.MarketEvent:
            chain = event.chain
            current_date = event.timestamp
            
            # Simple daily trading logic
            if current_date == self.last_trade_date:
                return
            
            # Exit logic: after 7 days
            positions = self.portfolio.get_positions()
            if positions:
                self.days_held = getattr(self, 'days_held', 0) + 1
                if self.days_held >= 7:
                    for symbol, qty in positions.items():
                        if qty != 0:
                            self.push_signal(bt.SignalEvent(current_date, symbol, bt.SignalType.Exit, abs(qty)))
                    self.days_held = 0
                return

            expiries = chain.get_available_expiries()
            if not expiries: return
            
            target_expiry = expiries[0]
            spot = chain.get_underlying_price()
            
            atm_call = chain.get_atm(bt.OptionType.Call, target_expiry)
            
            # Target ~5% OTM
            target_otm_strike = spot * 1.05
            otm_contracts = chain.get_contracts(bt.OptionType.Call, target_expiry)
            if not otm_contracts: return
            otm_call = min(otm_contracts, key=lambda c: abs(c.strike - target_otm_strike))

            if atm_call and otm_call:
                self.push_signal(bt.SignalEvent(current_date, atm_call.symbol, bt.SignalType.Long, 1))
                self.push_signal(bt.SignalEvent(current_date, otm_call.symbol, bt.SignalType.Short, 1))
                self.last_trade_date = current_date

if __name__ == "__main__":
    config = bt.BrokerConfig()
    config.commission = lambda qty, price, mult: 1.0 # $1 per trade (can be some added calc)
    config.slippage = lambda price: 0.01 # 1 cent slippage (can be some added calc)
    
    strategy = BullCallSpread()
    engine = bt.Backtester(50000.0, strategy, config) #setting initial capital, broker config and strat
    
    # Databento URL data fetching
    engine.add_data(bt.DataSourceType.URL, "databento:NVDA:2024-01-01:2024-04-01")
    engine.run() # for initiating backtesting (using engine.run before ingesting data will simply result in 0 trades executed)
```

## Core Architecture & Concepts
The backtester is built upon a standard **Event-Driven model**. This means the simulation progresses by consuming events from a single event queue.

### Main Classes
1. **`Backtester`**: The main engine orchestrator. It manages the initial capital, handles the data feeds, and runs the core event loop.

2. **`Strategy`**: The base class for all user-defined strategies. It exposes crucial methods like `on_event(event)` for processing market data and `push_signal(signal)` for executing trade intentions.

3. **`OptionChain` & `OptionContract`**: Represents the snapshot of available options at a given timestamp and the individual option contracts. They provide intuitive helper methods such as `get_atm()` and `get_contracts()`.

4. **`BrokerConfig`**: Responsible for simulating real-world trading costs, allowing users to define custom slippage and commission functions. If not set will simply result in an exception

5. **`Portfolio`**: Tracks the ongoing performance/state of the strategy, managing positions, current capital, and calculating PnL continuously.
